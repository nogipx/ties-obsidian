# Ties indexer

Автономный индексер эмбеддингов для плагина [Ties](../README.md). Считает эмбеддинги заметок вульта через Ollama и пишет `ties-embeddings.bin` в формате, **байт-совместимом с плагином** (тот же вход, тот же FNV-хэш, тот же бинарный layout, сортировка по пути).

Задуман как **единственный писатель** кэша на сервере: rhyolite CLI держит на сервере копию вульта и раздаёт готовый `.bin` на устройства, где плагин Ties работает в режиме «Кэш только для чтения».

## Требования

- Node.js 18+ (нужен глобальный `fetch`).
- Запущенный Ollama с моделью эмбеддингов (та же, что на устройствах, напр. `bge-m3`).
- Нет npm-зависимостей — один файл `ties-indexer.mjs`.

## Доставка на сервер

Файл вшит в плагин. В настройках Ties нажми **«Разложить индексер»** (или включи «Кэш только для чтения») — плагин запишет `ties-indexer.mjs` в папку кэша. Через rhyolite он попадёт на сервер, где остаётся только запустить его.

## Запуск

```bash
# один проход (инкрементально: считает только изменённые заметки)
node ties-indexer.mjs

# следить и обновлять по интервалу
node ties-indexer.mjs --watch
```

## Конфиг (env)

| Переменная | По умолчанию | Назначение |
|---|---|---|
| `TIES_VAULT` | родитель папки скрипта | корень вульта |
| `TIES_CACHE` | папка скрипта | куда писать `ties-embeddings.bin` |
| `TIES_OLLAMA` | `http://127.0.0.1:11434` | адрес Ollama |
| `TIES_MODEL` | `bge-m3` | модель эмбеддингов |
| `TIES_INTERVAL` | `30` | период watch, сек |
| `TIES_MAXCHARS` | `4000` | срез тела заметки |

Если положить скрипт в синхронизируемую папку кэша внутри вульта (напр. `<vault>/ties/`),
то дефолты сойдутся сами: `VAULT` = родитель, `CACHE` = эта папка. Тогда на сервере
достаточно `node ties-indexer.mjs --watch`.

## Как это устроено

- Инкрементально: читает существующий `.bin`, пропускает заметки с тем же хэшем содержимого и той же моделью, удаляет записи исчезнувших файлов.
- Пишет атомарно (`.tmp` + `rename`), чтобы rhyolite не подхватил недописанный файл.
- Ключ записи — путь заметки относительно корня вульта (совпадает с `file.path` в Obsidian).

## Синхронизация (rhyolite)

- `.bin` — один бинарный `LwwRegister`: при единственном писателе (этот индексер) гонок нет.
- Устройства держат плагин в режиме **read-only**, чтобы не конкурировать за запись.
- Стабильная сортировка по пути → CDC-дельты переносят только изменившиеся куски.
