# Ties

Плагин для Obsidian: **ты решаешь связь — плагин делает механику**. Он подсказывает похожие по смыслу заметки, пишет типизированную связь во `frontmatter` и показывает связи заметки. Тело заметки остаётся чистой прозой.

Идея — снять трение с ведения Zettelkasten: не писать ссылки руками, а выбирать «что с чем связать» кнопкой, а типы, направление и MOC-членство хранить в свойствах, не засоряя текст.

## Возможности

- **Типизированные связи во frontmatter.** Связь = свойство-ссылка с типом: `развивает: "[[X]]"`. Тело не трогается. Бэклинки и обновление при переименовании работают нативно.
- **Похожие по смыслу.** Локальные эмбеддинги через [Ollama](https://ollama.com) ранжируют кандидатов на связывание — сверху самые близкие по содержанию.
- **MOC как связь, а не автогенерация.** Членство в карте — это `moc: "[[MOC]]"`. Можно выборочно дописать пояснение «почему в карте» (хранится как алиас ссылки).
- **Просмотр связей.** Иконка в шапке заметки открывает модалку со связями (исходящие/входящие, у MOC — участники и орбита). Или встрой блок в тело:
  ````
  ```ties
  ```
  ````
- **Управление без правки текста.** Сменить тип связи (⇄), убрать связь (× с подтверждением), добавить участника в карту — всё из рендера.
- **Графовые инструменты.**
  - *Пути между заметками* — топ-5 кратчайших цепочек связей (алгоритм Йена), с промежуточными заметками.
  - *Все пути из заметки (обход)* — дерево всех достижимых заметок (каждая один раз), сворачиваемое.
  - *Окрестность (PageRank)* — что «вокруг» идеи с учётом всей структуры графа.
- **Автоштамп `created`** (дата-время) для новых заметок — без шаблонов.

## Требования

- Obsidian **1.12+**.
- Для «похожих по смыслу» — запущенный **Ollama** с моделью эмбеддингов (по умолчанию `bge-m3`). Без Ollama связывание работает, но список кандидатов — обычный поиск, без ранжирования по смыслу.

## Установка

### Вручную (из релиза)

1. Скачай `main.js`, `manifest.json`, `styles.css` из [релизов](https://github.com/nogipx/ties-obsidian/releases).
2. Положи их в `<vault>/.obsidian/plugins/ties/`.
3. Включи плагин в Settings → Community plugins.

### Через BRAT

Добавь репозиторий `nogipx/ties-obsidian` в [BRAT](https://github.com/TfTHacker/obsidian42-brat).

## Быстрый старт

1. (Опционально) Установи Ollama и модель: `ollama pull bge-m3`. Затем в плагине выполни команду **«Ties: Переиндексировать похожие»**.
2. Открой заметку, нажми иконку-звено в шапке (или `Cmd/Ctrl+Shift+L`) → **«＋ связать»**.
3. Выбери заметку из списка (похожие сверху, есть превью) → выбери тип связи.
4. Связь записана во frontmatter текущей заметки и видна в модалке/блоке.

На MOC-заметке кнопка называется **«＋ участник»** и работает наоборот: выбранная заметка вступает в карту.

## Как хранятся связи

В frontmatter заметки:

```yaml
---
развивает:
  - "[[Атомарность заметки]]"
контекст:
  - "[[Как я веду vault]]"
moc:
  - "[[MOC Zettelkasten|потому что это про метод]]"
---
```

- Тип связи = имя свойства.
- Направление: `развивает: [[X]]` читается как «эта заметка развивает X».
- `moc` — системный тип (членство в карте), его нельзя удалить/переименовать.
- Алиас у `moc`-ссылки — это пояснение «почему заметка в карте».

## Ollama (похожие по смыслу)

```bash
# установить модель эмбеддингов
ollama pull bge-m3
```

В настройках плагина задаётся URL Ollama (по умолчанию `http://localhost:11434`) и модель. Эмбеддинги кэшируются на диске и обновляются автоматически при правке заметки (можно выключить).

## Команды

| Команда | Что делает |
|---|---|
| Связать заметку | Похожие → тип → запись во frontmatter |
| Показать связи заметки | Модалка связей текущей заметки (`Cmd/Ctrl+Shift+L`) |
| Пути между заметками | Топ-5 кратчайших цепочек до выбранной заметки |
| Все пути из заметки (обход) | Дерево достижимых заметок |
| Окрестность заметки (PageRank) | Ранжированное соседство по графу |
| Переиндексировать похожие | Полный пересчёт эмбеддингов |
| Проверить Ollama | Диагностика связи с Ollama |
| Открыть боковую панель связей | Панель связей активной заметки |
| Справка | Встроенная справка по плагину |

## Настройки

- **Типы связей** — имя + описание, редактируемый список. Переименование типа мигрирует ключ во всех заметках.
- **Паттерн MOC** — как опознавать карты по имени файла (напр. `MOC *`); плюс `kind: moc` во frontmatter.
- **Кнопка связей в шапке заметки** — вкл/выкл.
- **Дата создания** — автоштамп `created`, свойство и формат (токены moment).
- **Похожие (Ollama)** — URL, модель, авто-обновление при правке.

## Разработка

```bash
npm install
npm run dev    # сборка в watch-режиме
npm run build  # проверка типов + прод-сборка
```

Для проверки в реальном vault: положи (или симлинкни) папку плагина в `<vault>/.obsidian/plugins/` и включи плагин.
